# Coding Standards

## General Principles
- Follow clean architecture and separation of concerns.
- Keep modules small, focused, and independently testable.
- Prefer explicit configuration over hardcoded values.
- Use descriptive names and avoid duplication.

## JavaScript Style
- Use ES modules.
- Prefer const and let over var.
- Use async/await for asynchronous code.
- Handle errors explicitly and return meaningful messages.

## Testing
- Add tests for every new public behavior.
- Prefer integration tests for HTTP endpoints.
- Keep tests deterministic and isolated.

## Documentation
- Document public APIs, services, and operational workflows.
- Update docs when architecture or process changes.
